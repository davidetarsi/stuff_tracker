// Flutter + Riverpod Project Rules (Optimized)

// Architecture
- Feature-first organization
- Riverpod 2.x with riverpod_annotation
- Freezed 3.x for models
- Drift for local database
- GoRouter for navigation

// Code Style
- Use const constructors when possible
- Prefer composition over inheritance
- Keep widgets small and focused
- Use Material 3 design system
- Responsive spacing/sizing via extensions

// Build
Run after model/provider changes:
dart run build_runner build --delete-conflicting-outputs

// State Management
- ONE provider per model
- Provider → Repository → DAO pattern
- Use @Riverpod(keepAlive: true) for data
- AsyncValue.when() for async states

// Common Patterns
- ErrorRetryDialog.executeWithRetry() for DB ops with retry
- context.go('/path') for navigation
- ItemCategory enum (not strings)
- TripItem = immutable snapshot
- ItemModel = mutable entity

//Cursor IDE Rules
- use only 1 worktree and don't generate any other worktree than the principal

//Update Rules
- When you modify some files, remember always to update the same files tests 
(if they exist) and the file ARCHITECTURE.md with corresponding changes

// Document creation Rules
- Don't create documentation .md if it's not requested by the user
